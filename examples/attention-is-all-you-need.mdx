## What this paper contributes

The Transformer is an encoder-decoder architecture for sequence transduction that replaces recurrence and convolutions entirely with multi-head self-attention. The core contributions are:

- Scaled dot-product attention (§3.2.1, Eq. 1)
- Multi-head attention (§3.2.2)
- Sinusoidal positional encoding (§3.5)
- The encoder and decoder stacks of N=6 layers (§3.1)
- Weight tying between embeddings and output projection (§3.4)

paper2code identifies this as **type a** (architecture paper) and generates the full model architecture, the label-smoothed loss, and a minimal training example — it does not attempt to generate a full WMT training pipeline.

    max_seq_len: int = 5000     # [UNSPECIFIED] — max length for positional encoding
    norm_eps: float = 1e-6      # [UNSPECIFIED] — LayerNorm epsilon not stated. Alternatives: 1e-5 (PyTorch default)

### Scaled dot-product attention (§3.2.1, Eq. 1)

```python
def scaled_dot_product_attention(
    query: torch.Tensor,
    key: torch.Tensor,
    value: torch.Tensor,
    mask: Optional[torch.Tensor] = None,
    dropout: Optional[nn.Dropout] = None,
) -> torch.Tensor:
    """§3.2.1, Eq. 1 — Attention(Q, K, V) = softmax(QK^T / √d_k) V

    Args:
        query: (batch, n_heads, seq_q, d_k)
        key:   (batch, n_heads, seq_k, d_k)
        value: (batch, n_heads, seq_k, d_v)
        mask:  (batch, 1, seq_q, seq_k) or (1, 1, seq_q, seq_k)
        dropout: optional dropout on attention weights

    Returns:
        (batch, n_heads, seq_q, d_v) — attention output
    """
    d_k = query.size(-1)

    # §3.2.1 — divide by √d_k, not √d_model — common mistake
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)

    if mask is not None:
        # §3.2.3 — set to −∞ before softmax, not multiply after
        scores = scores.masked_fill(mask == 0, float("-inf"))

    attn_weights = F.softmax(scores, dim=-1)

    if dropout is not None:
        attn_weights = dropout(attn_weights)

    return torch.matmul(attn_weights, value)
```

### Positional encoding (§3.5)

```python
class PositionalEncoding(nn.Module):
    """§3.5 — Positional Encoding (sinusoidal).

    PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
    PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
    """

    def __init__(self, config: TransformerConfig):
        super().__init__()
        self.dropout = nn.Dropout(config.dropout)

        pe = torch.zeros(config.max_seq_len, config.d_model)
        position = torch.arange(0, config.max_seq_len).unsqueeze(1).float()
        div_term = torch.exp(
            torch.arange(0, config.d_model, 2).float()
            * (-math.log(10000.0) / config.d_model)
        )

        pe[:, 0::2] = torch.sin(position * div_term)  # even indices: sin
        pe[:, 1::2] = torch.cos(position * div_term)  # odd indices: cos
        pe = pe.unsqueeze(0)  # (1, max_len, d_model) — batch broadcast
        self.register_buffer("pe", pe)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # §3.5 — "We apply dropout to the sums of the embeddings and the positional encodings"
        x = x + self.pe[:, :x.size(1)]
        return self.dropout(x)
```

### Encoder layer (§3.1)

```python
class EncoderLayer(nn.Module):
    """§3.1 — Single encoder layer.

    LayerNorm(x + Sublayer(x))  — post-norm formulation, matching Figure 1
    """

    def forward(
        self, x: torch.Tensor, src_mask: Optional[torch.Tensor] = None
    ) -> torch.Tensor:
        # §3.1 — Sub-layer 1: Multi-head self-attention
        # Post-norm: LayerNorm(x + Dropout(Sublayer(x)))
        attn_out = self.self_attn(x, x, x, mask=src_mask)
        x = self.norm1(x + self.dropout1(attn_out))

        # §3.1 — Sub-layer 2: Feed-forward network
        ff_out = self.feed_forward(x)
        x = self.norm2(x + self.dropout2(ff_out))

        return x
```

### LR schedule (§5.3, Eq. 3)

```python
def get_transformer_lr_schedule(
    optimizer: optim.Optimizer,
    d_model: int,
    warmup_steps: int = 4000,
) -> optim.lr_scheduler.LambdaLR:
    """§5.3, Eq. 3 — lr = d_model^(-0.5) * min(step^(-0.5), step * warmup^(-1.5))"""

    def lr_lambda(step: int) -> float:
        step = max(step, 1)  # avoid division by zero at step 0
        return d_model ** (-0.5) * min(step ** (-0.5), step * warmup_steps ** (-1.5))

    return optim.lr_scheduler.LambdaLR(optimizer, lr_lambda)
```

### Label-smoothed loss (§5.4)

```python
class LabelSmoothedCrossEntropy(nn.Module):
    """§5.4 — Label smoothing with KL divergence, ε_ls = 0.1.

    NOTE: Differs slightly from PyTorch's built-in label_smoothing parameter.
    PyTorch distributes ε/V to all classes; the paper distributes to incorrect
    classes only. For large vocabularies (V >> 1) the difference is negligible.
    We implement the paper's stated formulation.
    """

    def forward(self, logits: torch.Tensor, targets: torch.Tensor) -> torch.Tensor:
        vocab_size = logits.size(-1)
        logits = logits.contiguous().view(-1, vocab_size)
        targets = targets.contiguous().view(-1)

        log_probs = F.log_softmax(logits, dim=-1)
        nll_loss = -log_probs.gather(dim=-1, index=targets.unsqueeze(-1)).squeeze(-1)
        smooth_loss = -log_probs.mean(dim=-1)

        # §5.4 — combined loss
        loss = self.confidence * nll_loss + self.smoothing * smooth_loss

        non_pad_mask = targets != self.pad_idx
        loss = loss * non_pad_mask.float()
        return loss.sum() / non_pad_mask.float().sum().clamp(min=1.0)
```

**Debugging tips from the generated notes:**

- **Loss not decreasing** — verify the LR schedule follows Eq. 3 exactly. A constant LR or generic warmup schedule gives different results.
- **NaN in attention** — the mask must add `-inf` before softmax, not multiply after. All-zero rows (fully masked positions) cause NaN — ensure at least one position is always attended to.
- **Poor translation quality** — verify label smoothing (ε=0.1 with KL divergence), the √d_model embedding scale, and that weight tying is enabled.
- **Parameter count mismatch** — the base model should have approximately 65M parameters.

- Positional encoding matches §3.5 exactly — sinusoidal with correct div_term formula
- Encoder and decoder stacks are N=6 layers as specified in Table 3

**Citation anchoring**

**Config file**

- Label smoothing ε=0.1 from §5.4

### Discussion points

**Pre-norm vs post-norm** — The implementation uses post-norm (LayerNorm after residual addition), matching Figure 1. Some researchers argue the paper describes pre-norm in the text. The choice is reasonable and documented in REPRODUCTION_NOTES.md, but could be more explicit in the code comments.

**Dropout placement** — The paper says "Residual Dropout" in §5.4, applied "to the output of each sub-layer." The exact placement across attention weights, attention output, FFN internal, FFN output, and embeddings is somewhat ambiguous. The implementation handles this reasonably but could flag the attention-weight dropout more specifically as a partial specification.

**Weight tying** — Three-way weight tying (source embedding, target embedding, output projection) as described in §3.4 is correctly implemented. This is a detail many reimplementations miss.

**Training scope** — Correctly identified this as an architecture paper and limited the training code to a minimal example. A full WMT pipeline is out of scope and would obscure the contribution.

### What could be improved

1. The walkthrough notebook would benefit from explicit sanity checks on intermediate tensor shapes — particularly showing attention weights sum to 1 after softmax.
2. The `evaluate.py` could note that the original paper likely used `multi-bleu.perl`, which gives slightly different numbers from `sacrebleu`.
3. The training script could note that Table 2 numbers use beam search with beam_size=4 and length_penalty=0.6 (§6.1).

### Overall assessment

**Quality: High.** The implementation is citation-anchored, honest about uncertainty, and correctly scoped. A researcher can use this as a starting point with high confidence that the architecture matches the paper. The REPRODUCTION_NOTES.md is genuinely useful — it saves someone the work of auditing the paper themselves for unspecified details.

**Coverage: Core contribution fully implemented.** All architectural components from §3 are present and correct.

**Honesty: Good.** No silent assumptions. The review found no cases where the implementation confidently invented a detail the paper doesn't specify.
