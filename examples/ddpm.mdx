## What this paper contributes

DDPM introduces a training and sampling procedure for generative models based on reversing a gradual Gaussian noising process. The core contributions are:

- The simplified training objective L_simple (§3.4, Eq. 14, Algorithm 1)
- The reverse sampling algorithm (Algorithm 2)
- The connection between the simplified objective, variational inference, and denoising score matching
- The linear noise schedule β₁=10⁻⁴ to β_T=0.02 (§4)

The U-Net architecture is **not** the paper's contribution — it is adapted from PixelCNN++ and score matching prior work. paper2code identifies this correctly and marks U-Net implementation details that come from the official code rather than the paper text.

### Noise schedule (§4)

```python
def linear_noise_schedule(
    timesteps: int = 1000,
    beta_start: float = 1e-4,
    beta_end: float = 0.02,
) -> Dict[str, torch.Tensor]:
    """§4 — Linear variance schedule.

    "We set the forward process variances to constants increasing linearly
     from β_1 = 10^-4 to β_T = 0.02."

    Precomputes all quantities needed for training and sampling.
    """
    # §4 — linear schedule
    betas = torch.linspace(beta_start, beta_end, timesteps)

    # §2 — α_t = 1 - β_t
    alphas = 1.0 - betas

    # §2 — α̅_t = Π_{s=1}^{t} α_s  (cumulative product)
    alphas_cumprod = torch.cumprod(alphas, dim=0)

    # α̅_{t-1} — needed for posterior q(x_{t-1} | x_t, x_0)
    alphas_cumprod_prev = torch.cat([torch.tensor([1.0]), alphas_cumprod[:-1]])

    # Precompute for q(x_t | x_0) = N(x_t; √α̅_t x_0, (1-α̅_t)I)
    sqrt_alphas_cumprod = torch.sqrt(alphas_cumprod)
    sqrt_one_minus_alphas_cumprod = torch.sqrt(1.0 - alphas_cumprod)

    # §3.2, Eq. 7 — Posterior variance β̃_t = β_t * (1 - α̅_{t-1}) / (1 - α̅_t)
    posterior_variance = betas * (1.0 - alphas_cumprod_prev) / (1.0 - alphas_cumprod)

    return {
        "betas": betas,
        "alphas_cumprod": alphas_cumprod,
        "sqrt_alphas_cumprod": sqrt_alphas_cumprod,
        "sqrt_one_minus_alphas_cumprod": sqrt_one_minus_alphas_cumprod,
        "sqrt_recip_alphas": 1.0 / torch.sqrt(alphas),
        "posterior_variance": posterior_variance,
        # ... (additional precomputed quantities)
    }
```

### Forward process — q(x_t | x_0) in closed form (§3, Eq. 4)

```python
def q_sample(
    x_0: torch.Tensor,
    t: torch.Tensor,
    schedule: Dict[str, torch.Tensor],
    noise: Optional[torch.Tensor] = None,
) -> torch.Tensor:
    """§3, Eq. 4 — Forward process: sample x_t from q(x_t | x_0).

    "A notable property is that we can sample x_t at any arbitrary time step t
     in closed form: q(x_t | x_0) = N(x_t; √α̅_t x_0, (1 - α̅_t)I)"

    x_t = √α̅_t * x_0 + √(1 - α̅_t) * ε, where ε ~ N(0, I)
    """
    if noise is None:
        noise = torch.randn_like(x_0)

    sqrt_alpha_cumprod = schedule["sqrt_alphas_cumprod"][t].view(-1, 1, 1, 1)
    sqrt_one_minus_alpha_cumprod = schedule["sqrt_one_minus_alphas_cumprod"][t].view(-1, 1, 1, 1)

    # §3, Eq. 4
    return sqrt_alpha_cumprod * x_0 + sqrt_one_minus_alpha_cumprod * noise
```

### Simplified training objective — L_simple (§3.4, Eq. 14)

```python
class DDPMLoss(nn.Module):
    """§3.4, Eq. 14 — Simplified training objective L_simple.

    Computes MSE between true noise and predicted noise:
        L = ||ε − ε_θ(x_t, t)||²

    "We found it beneficial to sample quality (and simpler to implement)
    to train on the following variant of the variational bound... L_simple"
    """

    def forward(
        self,
        noise_pred: torch.Tensor,
        noise_true: torch.Tensor,
    ) -> torch.Tensor:
        """
        §3.4, Eq. 14 — L_simple = E[||ε − ε_θ(x_t, t)||²]

        Args:
            noise_pred: (batch, C, H, W) — predicted noise ε_θ(x_t, t)
            noise_true: (batch, C, H, W) — true noise ε ~ N(0, I)
        """
        # §3.4 — simple MSE, "equivalent to (a re-weighted variant of) the ELBO"
        return nn.functional.mse_loss(noise_pred, noise_true)
```

### Reverse sampling — Algorithm 2

```python
@torch.no_grad()
def p_sample(
    model: nn.Module,
    x_t: torch.Tensor,
    t: torch.Tensor,
    t_index: int,
    schedule: Dict[str, torch.Tensor],
) -> torch.Tensor:
    """Algorithm 2, lines 3-4 — Single reverse step: x_{t-1} from p_θ(x_{t-1} | x_t).

    "x_{t-1} = 1/√α_t * (x_t - β_t/√(1-α̅_t) * ε_θ(x_t, t)) + σ_t * z"
    where z ~ N(0, I) if t > 1, else z = 0.
    """
    predicted_noise = model(x_t, t)

    beta_t = schedule["betas"][t_index]
    sqrt_recip_alpha_t = schedule["sqrt_recip_alphas"][t_index]
    sqrt_one_minus_alpha_cumprod_t = schedule["sqrt_one_minus_alphas_cumprod"][t_index]

    # Algorithm 2, line 4 — μ_θ = 1/√α_t * (x_t - β_t/√(1-α̅_t) * ε_θ(x_t, t))
    mean = sqrt_recip_alpha_t * (
        x_t - beta_t / sqrt_one_minus_alpha_cumprod_t * predicted_noise
    )

    if t_index == 0:
        # Algorithm 2, line 3 — z = 0 at final step
        return mean
    else:
        # §3.4 — σ²_t = β_t (fixed small variance)
        sigma_t = torch.sqrt(schedule["betas"][t_index])
        noise = torch.randn_like(x_t)
        return mean + sigma_t * noise


@torch.no_grad()
def sample(
    model: nn.Module,
    schedule: Dict[str, torch.Tensor],
    image_shape: tuple,
    device: torch.device,
) -> torch.Tensor:
    """Algorithm 2 — Full reverse sampling process.

    1: x_T ~ N(0, I)
    2: for t = T, ..., 1 do
    3:   z ~ N(0, I) if t > 1, else z = 0
    4:   x_{t-1} = 1/√α_t (x_t - β_t/√(1-α̅_t) ε_θ(x_t, t)) + σ_t z
    5: end for
    6: return x_0
    """
    model.eval()
    timesteps = len(schedule["betas"])
    x = torch.randn(image_shape, device=device)  # Algorithm 2, line 1: x_T ~ N(0, I)

    for t_index in reversed(range(timesteps)):
        t = torch.full((image_shape[0],), t_index, device=device, dtype=torch.long)
        x = p_sample(model, x, t, t_index, schedule)

    return x
```

### EMA (§4)

```python
class EMA:
    """§4 — Exponential Moving Average of model parameters.

    "We report sample quality metrics using an exponential moving average (EMA)
     of model parameters with a decay factor of 0.9999."

    The EMA weights are used for sampling/evaluation, not for training.
    """

    def __init__(self, model: nn.Module, decay: float = 0.9999):
        self.decay = decay
        self.shadow = {}
        for name, param in model.named_parameters():
            if param.requires_grad:
                self.shadow[name] = param.data.clone()

    @torch.no_grad()
    def update(self, model: nn.Module):
        """Update EMA weights after each training step."""
        for name, param in model.named_parameters():
            if param.requires_grad and name in self.shadow:
                self.shadow[name].mul_(self.decay).add_(
                    param.data, alpha=1.0 - self.decay
                )
```

### Sinusoidal timestep embedding (§3.3)

```python
class SinusoidalTimeEmbedding(nn.Module):
    """§3.3 — "The diffusion time t is specified by adding the Transformer
    sinusoidal position embedding into each residual block."

    [FROM_OFFICIAL_CODE] The embedding dimension and MLP projection follow
    the official implementation.
    """

    def forward(self, t: torch.Tensor) -> torch.Tensor:
        """
        Args:
            t: (batch,) — integer timesteps
        Returns:
            (batch, embed_dim) — sinusoidal embedding
        """
        half_dim = self.embed_dim // 2
        emb = math.log(10000.0) / (half_dim - 1)
        emb = torch.exp(torch.arange(half_dim, device=t.device, dtype=torch.float32) * -emb)
        emb = t.float().unsqueeze(1) * emb.unsqueeze(0)
        emb = torch.cat([torch.sin(emb), torch.cos(emb)], dim=-1)  # (batch, embed_dim)
        return emb
```

### Residual block with time conditioning (§3.3, Appendix B)

```python
class ResidualBlock(nn.Module):
    """§3.3 — "group normalization throughout... Transformer sinusoidal position
    embedding into each residual block"

    [FROM_OFFICIAL_CODE] Structure: GroupNorm -> SiLU -> Conv -> GroupNorm -> SiLU -> Dropout -> Conv + residual
    """

    def forward(self, x: torch.Tensor, t_emb: torch.Tensor) -> torch.Tensor:
        """
        Args:
            x: (batch, in_channels, H, W)
            t_emb: (batch, time_embed_dim)
        Returns:
            (batch, out_channels, H, W)
        """
        h = self.norm1(x)
        h = F.silu(h)
        h = self.conv1(h)

        # Add time embedding — broadcast over spatial dimensions
        t = self.time_proj(F.silu(t_emb))
        h = h + t.unsqueeze(-1).unsqueeze(-1)  # (batch, out_channels, H, W)

        h = self.norm2(h)
        h = F.silu(h)
        h = self.dropout(h)
        h = self.conv2(h)

        return h + self.skip(x)  # residual connection
```

**Debugging tips from the generated notes:**

- **Samples look like noise after training** — verify the noise schedule. α̅_T should be close to 0. If not, the forward process doesn't destroy enough information and sampling from N(0,1) doesn't work.
- **Samples are blurry** — DDPM needs 800k+ training steps on CIFAR-10. Also verify you are using EMA weights for sampling, not training weights.
- **Mode collapse or repetitive samples** — check that the timestep embedding is working. If t is not properly conditioned, the model can't distinguish noise levels.
- **Loss doesn't decrease** — verify the forward process: x_t = √(α̅_t) * x_0 + √(1 - α̅_t) * ε. Check signs and square roots.
- **NaN in sampling** — at t=0, some terms can be 0/0. The original code clips or special-cases t=0.
