## Install the skill

Run the following command to add paper2code to your Claude Code setup:

```bash
npx skills add PrathamLearnsToCode/paper2code/skills/paper2code
```

You will be prompted to make three choices:

<Steps>
  <Step title="Select agents">
    Choose which coding agents to enable the skill for. Select **Claude Code** (recommended) or any other agents you use.
  </Step>
  <Step title="Choose scope">
    Select **Global** (recommended) to use the skill across all your projects, or **Project** to limit it to the current directory.
  </Step>
  <Step title="Choose method">
    Select **Symlink** (recommended) so the skill stays up to date automatically, or **Copy** to take a snapshot.
  </Step>
</Steps>

## Run your first implementation

Open Claude Code and use the `/paper2code` command with any arXiv URL or paper ID:

```bash
claude
```

Then inside the agent:

```
/paper2code https://arxiv.org/abs/1706.03762
```

The skill fetches the paper, audits every implementation detail, and generates a citation-anchored Python implementation in the current directory.

## Usage examples

<CodeGroup>

```bash Basic
/paper2code https://arxiv.org/abs/1706.03762
```

```bash Bare arxiv ID
/paper2code 1706.03762
```

```bash Specify framework
/paper2code https://arxiv.org/abs/2006.11239 --framework jax
```

```bash Full mode (training loop + data pipeline)
/paper2code 2106.09685 --mode full
```

```bash Educational mode (extra comments + paper guide)
/paper2code https://arxiv.org/abs/2010.11929 --mode educational
```

</CodeGroup>

## What happens next

When the skill completes, it prints a summary:

```
✓ paper2code complete for: Attention Is All You Need
  Output directory: attention_is_all_you_need/
  Files generated: src/model.py, src/loss.py, src/data.py, src/train.py,
                   src/evaluate.py, src/utils.py, configs/base.yaml,
                   notebooks/walkthrough.ipynb, REPRODUCTION_NOTES.md, README.md
  Unspecified choices: 7 (see REPRODUCTION_NOTES.md)
  Mode: minimal | Framework: pytorch
```

<Tip>
  Read `REPRODUCTION_NOTES.md` first. It lists every implementation choice the paper did not specify, so you know exactly what to verify before using the code for research.
</Tip>
